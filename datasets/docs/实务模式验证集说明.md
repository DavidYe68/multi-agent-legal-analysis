# ICAD 实务模式验证集说明

## 1. 目的

该验证集用于检查同一套ICAD推理内核能否在实务模式下，把上游的争点、控辩论证、法官未决事项和评议结果，重新组织为“风险提示—关键证据缺口—下一步调查建议—结论边界”。它不是新的案件库，也不改变原有12案Benchmark；它从原有案例中选取4个代表性案件作为实务输出验证子集。

## 2. 为什么不复制案件

教学模式与实务模式的区别主要发生在输出层，而不是事实层。同一案件在教学模式下强调推理过程、正反论证和课堂问题，在实务模式下强调风险等级、补证顺序、行动建议和不可越过的结论边界。因此，运行时应通过task_mode覆盖来切换Writer，而不应复制一套事实完全相同的新案例。

## 3. 验证子集

- criminal_001a：主观明知证据不足，测试能否给出补充电子数据、报酬记录和同案人员印证的行动建议。
- criminal_001b：主观明知证据较强，测试能否提高风险等级，同时保留电子数据程序核验这一前提。
- criminal_003a：伤害因果关系未闭合，测试能否把补充鉴定和关键过程证据列为最高优先级。
- criminal_006a：供述可采性存疑，测试能否区分取证合法性、供述真实性和排除供述后的剩余证据充分性。

## 4. 文件用途

`practice_validation.json`只列出需要运行的案例ID。`practice_gold/`中的文件是实务模式专用评价标准，只能由评估脚本在模型运行结束后读取，不能进入任何Prompt、state、Agent输入或前端Trace。

## 5. 最低通过标准

每个实务报告必须满足：`mode_hint`为`practice`；`risk_flags`和`next_step_suggestions`均非空；风险与建议能够追溯到已有争点、证据或证据缺口；不得编造新事实；不得将开放争点写成确定结论；前端切换到实务模式时必须读取真实实务Writer输出，而不是仅改变教学报告的页面排版。
